POSIX Utilities Package

  • Posted: Dec 23, 2004, Last updated: Apr 02 2005
  • Summary: A collection of basic C++ utility classes for inter-process message passing, shared memory, synchronization and thread management. This library is for UNIX systems.

Introduction

In implementing experiments in the lab during the course of my research, I found that I was using certain programming methods and data structures again and again. This is a collection of some of those essentials built into a single set of re-usable C++ classes. In developing this library, I have restricted myself to using features defined in the POSIX standard (PASC 1003.1, 1003.4), and hence the name of the library. I work both with Linux and QNX operating systems. Hence my priorities in developing these utilities were portability and performance, in that order. This library currently provides the following functionality.

  • Barrier: The pthread barrier synchronization object.
  • ErrnoException: A mechanism for basic run-time exception handling.
  • MessageQueue: Inter-process messaging mechanism.
  • RecursiveMutex: A wrapper for pthread mutex, with the added functionality that it is recursive.
  • RWLock: The pthread reader-writer lock.
  • ShMem: Shared memory objects.
  • StatusReport: An object for storing status messages containing a integer code, a message and a timestamp.
  • Thread: A mechanism to execute code in a separate thread.
  • TCPClient and TCPServer: A TCP/IP based client-server pair.
  • UDPClient and UDPServer: A UDP based client-server pair.

Installation

  • Download sources from here: putils.tgz.
  • Unzip and untar the archive, build the library as follows:
    tar zxvf putils.tgz
    cd putils
    make
  • Install the library. Login as root and
    make install
  • Try the example programs in the examples directory. Just run make in the example directory to create the executables. If any example fails to run, please let me know.

Applications

Programming for compter vision and robotics can be pretty complicated due to the need for simultaneous interaction between many different hardware and software components with real-time constraints. If used properly, these classes can take some of the pain away.
